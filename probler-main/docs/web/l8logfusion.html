Overview

L8LogFusion is a powerful distributed log aggregation and fusion system designed to collect, centralize, and browse logs from multiple sources in containerized environments. Built with Go and Protocol Buffers, it provides real-time log collection, intelligent buffering, and web-based browsing capabilities for modern cloud-native applications.

Deployment

Docker Images

L8LogFusion provides three production-ready Docker images for easy deployment:

Log Agent

Image: saichler/probler-logagent:latest

docker run -d \ -e NODE_IP=auto \ -e LOGPATH=/var/log \ -e LOGFILE="*" \ -v /var/log:/var/log:ro \ saichler/probler-logagent:latest

Log Server

Image: saichler/probler-logserver:latest

docker run -d \ -v /data/logdb:/data/logdb \ saichler/probler-logserver:latest 12443

Web UI

Image: saichler/probler-logsui:latest

docker run -d \ -p 26000:26000 \ -v /data/probler:/data/probler \ saichler/probler-logsui:latest

API Reference

REST Endpoints

The Log Server provides a comprehensive REST API for log operations:

POST /logs

Submit new log entries to the server. Accepts L8LogF protocol buffer messages.

GET /logs

Query stored logs with pagination support. Returns logs in 5KB pages.

PUT /logs

Update existing log entries. Replaces entire log records.

PATCH /logs

Merge log entries. Appends new logs to existing records.

DELETE /logs

Remove log entries. Deletes specified log records from storage.

Protocol Buffer Messages

// Main log fusion message message L8LogF { string sourceIp = 1; // IP of source machine string filename = 2; // Log file name repeated string logs = 3; // Array of log lines } // Log collector configuration message L8LogConfig { string path = 1; // Directory to monitor string name = 2; // Filename or "*" for all }

Usage Examples

Running the Log Collector

# Set environment variables export NODE_IP=192.168.1.100 export LOGPATH=/var/log export LOGFILE="*" # Collect all .log/.err files # Run the collector go run go/agent/logs/docker/main.go

Start collecting logs from the specified directory with automatic file discovery.

Starting the Log Server

# Start server on virtual network port 12443 go run go/agent/logserver/docker/main.go 12443 # Server will listen for incoming log streams # Logs stored at /data/logdb/{sourceIP}/{filename}

Launch the central server that receives and persists logs from all collectors.

Accessing the Web UI

# Start the web UI (port 26000) go run go/agent/ui/main.go # Access in browser http://localhost:26000 # Health check endpoint http://localhost:26000/health

Launch the web interface for browsing and querying collected logs.
