Overview

Layer 8 Topology is a high-performance network topology visualization and management system written in Go with an interactive web-based UI. It provides real-time network topology discovery, multiple layout algorithms, and GPU-accelerated WebGL rendering for handling large-scale network graphs.

Performance: 30,000+ nodes | 300,000+ links | WebGL provides 10-30x speedup over SVG

Usage Examples

Creating a Topology Service

import ( "github.com/saichler/l8topology/go/topo/topo_service" "github.com/saichler/l8types/go/types" ) // Create a new topology service service := topo_service.NewTopoService( resources, servicePoints, )

Initialize the core topology service with your resources.

Adding Nodes

// Create a topology node node := &l8topo.L8TopologyNode{ NodeId: "node-1", Name: "Core Router", Type: l8topo.L8TopologyNodeType_ROUTER, Location: "New York", } // Add node via service service.Post(elements, vnic)

Add network nodes with type and location information.

Creating Links

// Create a topology link link := &l8topo.L8TopologyLink{ LinkId: "link-1", Aside: "node-1", Zside: "node-2", Direction: l8topo.L8TopologyLinkDirection_Bidirectional, Status: l8topo.L8TopologyLinkStatus_Up, }

Connect nodes with links including direction and status.

Protocol Buffer Types

// Core topology container message L8Topology { string name = 1; map<string, L8TopologyNode> nodes = 2; map<string, L8TopologyLink> links = 3; map<string, L8TopologyLocation> locations = 4; } // Node types: Generic, Switch, Router, Firewall, LoadBalancer, Server, Storage, Cloud // Link directions: AsideToZside, ZsideToAside, Bidirectional // Link statuses: Up, Down, Partial // Layouts: Location, Hierarchical, Circular, Radial, Force_Directed
