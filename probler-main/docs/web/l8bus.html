Overview

The Layer 8 Network BUS provides a comprehensive virtual networking layer built on top of TCP connections, enabling distributed systems to communicate as if they were on a single network. Latest release (December 2025) introduces round robin load balancing, health participant tracking, multicast keep-alive, and enhanced service management.

Core Components

VNet (Virtual Network)

Location: vnet/VNet.go

Purpose: Central network switch that manages connections between nodes

  • TCP listener for incoming connections
  • Message routing and switching
  • Health monitoring integration
  • Service discovery
  • Connection management

VNic (Virtual Network Interface)

Location: vnic/VirtualNetworkInterface.go

Purpose: Network interface for individual nodes to connect to the overlay

  • Bidirectional communication (TX/RX)
  • Keep-alive mechanisms
  • Automatic reconnection
  • Service API integration

Protocol

Location: protocol/Protocol.go

Purpose: Message serialization/deserialization and protocol handling

  • Base64 encoded message data
  • Sequence number management
  • Message creation and parsing
  • Transaction state management

Key Features

Round Robin Load Balancing

Intelligent round robin service selection for optimized load distribution across service instances, ensuring fair resource utilization and improved reliability.

Health Participant Tracking

Enhanced health monitoring with participant tracking for comprehensive service visibility, enabling better observability and debugging capabilities.

Multicast Keep-Alive

Migrated keep-alive mechanism to multicast for improved network efficiency, reducing connection overhead and improving scalability.

Multicast Messaging

Full multicast messaging support enabling efficient one-to-many communication patterns for distributed systems, with optimized message forwarding and routing.

MapReduce Capabilities (MapR)

Integrated MapReduce functionality for distributed data processing, enabling efficient parallel computation across the network overlay.

External NIC Support

Enhanced support for external network interface cards, allowing seamless integration with diverse network configurations and environments.

Transaction Timeout Management

Configurable transaction timeout mechanisms with automatic cleanup and recovery for reliable distributed operations.

Intelligent Service Batching

Advanced service batch processing that groups related operations for improved throughput, reduced latency, and optimal resource utilization.

Network Discovery

Automatic peer discovery using UDP broadcasts with dynamic network topology building and IP-based routing decisions.

Message Routing

Service-based message routing with support for unicast and multicast, plus leader election for service instances.

Connection Management

Internal vs external connection classification, automatic reconnection on failures, and connection pooling.

Security

Connection validation, security provider integration, and token-based authentication.

Network Topology

Hybrid Network Architecture

The overlay creates a sophisticated hybrid network topology with three distinct connection types:

Internal Connections

Direct connections between processes on the same machine for high-performance local communication.

External Connections

TCP connections to remote VNet switches enabling cross-network communication.

Service Routing

Messages routed based on service names and areas, providing intelligent service-aware networking.

Usage Examples

Creating a VNet (Network Switch)

vnet := NewVNet(resources) err := vnet.Start() if err != nil { log.Fatal("Failed to start VNet:", err) }

Initialize and start the central network switch that will manage all node connections and route messages.

Creating a VNic (Network Interface)

vnic := NewVirtualNetworkInterface(resources, nil) vnic.Start() // Wait for connection to be established vnic.WaitForConnection()

Create a virtual network interface that connects your application to the Layer8 overlay network.

Service Registration with SLA

// Services now use SLA-based activation for better monitoring // Health service provides unified monitoring and discovery health.Activate(vnic, isVnet) // Service is registered with: // - SLA tracking enabled // - Primary key on "AUuid" // - Automatic health list aggregation

Register services with SLA tracking for automatic discovery and health monitoring.

Configuration

The overlay network is configured through SysConfig which includes:

  • Local and remote UUIDs and aliases
  • VNet port configuration
  • Service definitions
  • Queue sizes and data limits
