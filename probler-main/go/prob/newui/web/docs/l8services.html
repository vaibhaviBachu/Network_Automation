Overview

Layer 8 Services is a high-performance distributed services framework built on the Layer8 platform that provides enterprise-grade capabilities for modern distributed applications. Recently enhanced with Map-Reduce distributed processing pattern, SLA-based service activation architecture, improved error handling, and optimized transaction management, it delivers advanced API management, concurrency control, and intelligent caching for scalable distributed systems.

Advanced Distributed Cache

Thread-safe, synchronized distributed caching with persistent storage support, optimized data fetching, and automatic synchronization across nodes.

Enhanced Transaction Management

ACID transaction support across distributed services with improved 2-phase commit protocol, optimized state management, and enhanced coordination ensuring data consistency.

SLA-Based Service Management

Registration, activation, and lifecycle management using Service Level Agreements. Enhanced with improved error handling, crash prevention, and graceful failure recovery.

Intelligent Replication Services

Data replication and synchronization with deterministic round-robin load balancing. Features UUID-based tracking for true fair distribution across service replicas.

Real-time Notification System

Event-driven notifications for cache updates and service state changes with real-time propagation and enhanced reliability.

Comprehensive Security Integration

Built-in security checks for all service operations with authentication, authorization support, and advanced IP comparison utilities.

Map-Reduce Pattern Support

Distributed Map-Reduce processing for parallel data operations across service nodes. Enables efficient distributed computing with automatic result merging and aggregation.

Architecture

The framework is organized into several core components that work together to provide a robust distributed services platform:

Core Services

DCache (services/dcache/)

Distributed caching with thread-safe operations, persistence, and notifications. Supports automatic synchronization and conflict resolution.

Transaction Manager (services/transaction/)

Distributed transaction coordination with full ACID properties and optimized state management. Implements enhanced 2-phase commit protocol with improved performance and resource cleanup.

Service Manager (services/manager/)

Service lifecycle management and request routing. Handles service registration, discovery, and load balancing.

Replication Service (services/replication/)

Data synchronization across distributed nodes with configurable replication strategies and conflict resolution.

Key Features

  • Thread-Safe Operations: All cache and service operations are protected with appropriate locking mechanisms
  • Optimized Transaction Processing: Recently refactored transaction system with improved state management and coordination
  • Persistence Layer: Optional storage backend for cache durability
  • Event Notifications: Real-time notifications for data changes and service events
  • Security Integration: Built-in security checks for all service operations
  • Health Monitoring: Service health tracking and reporting
  • Web Service Integration: RESTful API endpoints for external service interaction

API Reference

Distributed Cache Operations

  • Put(key, value) - Store value in cache
  • Get(key) - Retrieve value from cache
  • Delete(key) - Remove value from cache
  • Update(key, value) - Update existing value
  • Sync() - Synchronize cache with peers
  • Collect(filter) - Bulk retrieve filtered items

Service Handler Interface

Services must implement the IServiceHandler interface:

  • Post(payload, vnic) - Create new resources
  • Put(payload, vnic) - Replace existing resources
  • Patch(payload, vnic) - Update existing resources
  • Delete(payload, vnic) - Remove resources
  • Get(payload, vnic) - Retrieve resources
  • Failed(payload, vnic, message) - Handle failure scenarios

Map-Reduce Operations

Distributed processing with the IMapReduceService interface:

  • MapReduce(handler, payload, action) - Execute distributed operation
  • PeerRequest(msg, vnic) - Send parallel requests to peers
  • Merge(results) - Aggregate results from distributed nodes
  • MapR_POST - Distributed create operation
  • MapR_GET - Distributed query operation
  • MapR_PUT/PATCH/DELETE - Distributed update operations
